## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

숙소 호스트들은 마치 **24시간 응답해야 하는 콜센터 직원**처럼 살고 있습니다.

- "와이파이 비밀번호가 뭐예요?"
- "주변에 맛집 추천해주세요!"
- "주차장은 어디예요?"

같은 질문이 손님마다 반복됩니다. 종이 안내문은 금방 더러워지고, 정보가 바뀌면 다시 인쇄해야 합니다. PC 앞에 앉아있을 시간이 없는 호스트는 이동 중에 급하게 정보를 수정해야 할 때 막막합니다.

게스트 입장에서도 1박 투숙을 위해 앱을 설치하라는 요구는 **결혼식 참석을 위해 정장을 사라는 것**만큼 부담스럽습니다.

### 1.2 목표 상황 (To-Be)

**"모바일 청첩장처럼 예쁘고, 링크 하나로 모든 정보가 전달되는 세상"**

- 호스트: 언제 어디서든 모바일로 안내서를 예쁘게 수정
- 게스트: 앱 설치 없이 URL 하나로 모든 숙소 정보 + AI 컨시어지 이용
- 반복 문의: AI가 24시간 자동 응대

### 1.3 왜 지금인가? (Why Now)

- **모바일 청첩장 시장의 성공**: 필카드, 필프레임 등의 성공으로 검증된 비즈니스 모델
- **숙박 시장의 변화**: 플랫폼(야놀자 등)에서 개별 숙소의 '브랜딩' 중심으로 이동
- **AI 기술 성숙**: RAG 기반 AI 챗봇으로 자동 응대가 현실적으로 가능해짐

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩/로그인)

- [ ] 호스트로서, 이메일 또는 소셜 로그인으로 간편하게 가입할 수 있어야 한다. 왜냐하면 가입 장벽을 낮춰야 서비스를 써볼 의향이 생기기 때문이다.
- [ ] 호스트로서, 로그인 상태가 유지되어야 한다. 왜냐하면 매번 로그인하는 것은 번거롭기 때문이다.
- [ ] 게스트로서, 로그인 없이 안내서를 볼 수 있어야 한다. 왜냐하면 1박 투숙객에게 회원가입은 과한 요구이기 때문이다.

### FEAT-1: 커스텀 에디터 (MVP)

- [ ] 호스트로서, 블록을 추가/삭제/순서 변경하여 안내서를 만들 수 있어야 한다. 왜냐하면 내 숙소에 맞게 구성해야 의미가 있기 때문이다.
- [ ] 호스트로서, PC와 모바일 모두에서 동일하게 에디터를 사용할 수 있어야 한다. 왜냐하면 이동 중에도 수정이 필요하기 때문이다.
- [ ] 호스트로서, 실시간 미리보기를 보면서 편집할 수 있어야 한다. 왜냐하면 결과물을 확인하며 작업해야 실수를 줄일 수 있기 때문이다.
- [ ] 호스트로서, 워터마크 포함 무료 미리보기를 할 수 있어야 한다. 왜냐하면 결제 전에 품질을 확인하고 싶기 때문이다.

### FEAT-2: 필수 정보 블록 (MVP)

- [ ] 호스트로서, 체크인/아웃 시간, Wi-Fi 비밀번호, 지도, 맛집 추천 블록을 추가할 수 있어야 한다. 왜냐하면 이것이 게스트가 가장 많이 묻는 정보이기 때문이다.
- [ ] 게스트로서, Wi-Fi 비밀번호를 클릭 한 번으로 복사할 수 있어야 한다. 왜냐하면 복잡한 비밀번호를 직접 입력하기 귀찮기 때문이다.
- [ ] 게스트로서, 지도 링크를 눌러 바로 네이버/카카오맵으로 이동할 수 있어야 한다. 왜냐하면 길 찾기가 편해야 하기 때문이다.

### FEAT-3: 테마/디자인 (MVP)

- [ ] 호스트로서, 프리셋 테마(감성/모던/내추럴)를 선택할 수 있어야 한다. 왜냐하면 디자인 감각이 없어도 예쁜 결과물을 원하기 때문이다.
- [ ] 호스트로서, 포인트 컬러와 폰트를 커스터마이징할 수 있어야 한다. 왜냐하면 내 숙소 브랜드와 어울리게 맞추고 싶기 때문이다.

### FEAT-4: AI 컨시어지 (MVP)

- [ ] 게스트로서, 안내서 내용을 기반으로 AI에게 질문할 수 있어야 한다. 왜냐하면 원하는 정보를 찾아 읽기보다 물어보는 게 빠르기 때문이다.
- [ ] 호스트로서, AI 컨시어지를 ON/OFF 할 수 있어야 한다. 왜냐하면 유료 기능으로 제어가 필요하기 때문이다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)

- **FEAT-1**: 커스텀 에디터 (PC/모바일 반응형, 블록 추가/삭제/순서변경)
- **FEAT-2**: 필수 정보 블록 5종 (히어로, 필수 정보, 와이파이, 지도, 맛집)
- **FEAT-3**: 테마/디자인 (프리셋 테마, 컬러/폰트 커스터마이징)
- **FEAT-4**: AI 컨시어지 (RAG 기반 자동 응대)
- **FEAT-0**: 기본 온보딩/로그인, 네이버 스마트스토어 결제 연동

### 5.2 옵션 (시간 허용 시)

- 호스트 대시보드 (조회수, 많이 클릭한 버튼 통계)
- 다국어 자동 번역 (영어/일어/중국어)
- QR코드 생성 (보딩패스 컨셉)

### 5.3 Non-goals (명시적 제외)

- 예약 관리 시스템 (야놀자/에어비앤비 연동 X)
- 결제 시스템 직접 구현 (네이버 스마트스토어 위임)
- 다크 모드 (v2에서 추가)
- 호스트 간 템플릿 공유/판매 마켓

## 7. 해결 가설과 실험

### 7.1 가설

> 만약 **모바일에서도 PC와 동일한 수준의 에디터 경험**을 제공하면,
> **호스트의 에디터 완료율이 60% 이상**이 될 것이다.
> 왜냐하면 이동이 잦은 호스트도 언제든 수정할 수 있기 때문이다.

### 7.2 실험 루프

```
실험 → 관측 → 학습 → 다음 가설
```

| 단계 | 내용 |
|------|------|
| 실험 | 모바일 에디터 프로토타입으로 5명 호스트 테스트 |
| 관측 | 에디터 완료율, 소요 시간, 만족도 설문 |
| 학습 | 완료율 60% 이상이면 성공, 미만이면 UX 개선 필요 |
| 다음 | AI 컨시어지 만족도 실험 |
