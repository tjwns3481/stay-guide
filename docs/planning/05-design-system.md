# Design System (기초 디자인 시스템) - Roomy

> 비기술자도 이해할 수 있는 디자인 언어로 작성합니다.
> 모바일 청첩장 스타일을 참고하여 깔끔하고 감성적인 디자인을 지향합니다.

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 호스트의 시간을 벌어주고 숙소의 브랜드 가치를 높여주는 운영 솔루션 |
| 2 | 페르소나 | 감성 숙소/펜션/에어비앤비 호스트 (디자인과 고객 경험 중시, 반복 문의에 지친 상태) |
| 3 | 핵심 기능 | FEAT-1: 커스텀 에디터 (블록형 안내서 제작/수정) |
| 4 | 성공 지표 (노스스타) | 무료 → 유료 라이선스 전환율 |
| 5 | 입력 지표 | 신규 가입 호스트 수, 안내서 조회 수 |
| 6 | 비기능 요구 | 모바일에서도 PC와 동일한 수준의 에디터 경험 제공 |
| 7 | Out-of-scope | 예약 관리, 결제 시스템 자체 구현, 다크 모드 |
| 8 | Top 리스크 | 모바일 에디터 UX가 복잡해져 진입장벽이 높아질 위험 |
| 9 | 완화/실험 | 5명 이상의 실제 호스트와 프로토타입 테스트 |
| 10 | 다음 단계 | 커스텀 에디터 코어 개발 시작 |

## 1. 디자인 철학

### 1.1 핵심 가치

| 가치 | 설명 | 구현 방법 |
|------|------|----------|
| **깔끔함** | 불필요한 요소 없이 정보에 집중 | 충분한 여백, 미니멀한 UI |
| **따뜻함** | 친근하고 환영받는 느낌 | 부드러운 컬러, 라운드 모서리 |
| **신뢰감** | 프로페셔널하고 세련된 인상 | 일관된 타이포그래피, 정돈된 레이아웃 |

### 1.2 참고 서비스 (무드보드)

| 서비스 | 참고할 점 | 참고하지 않을 점 |
|--------|----------|-----------------|
| 필카드/필프레임 | 블록형 에디터, 깔끔한 프리뷰 | - |
| 에어비앤비 | 숙소 정보 레이아웃, 지도 연동 | 복잡한 필터 UI |
| 노션 | 블록 드래그앤드롭, 명확한 계층 | 모노톤 컬러 |
| Apple | 부드러운 애니메이션, 세련된 버튼 | 과한 그라데이션 |

### 1.3 톤 & 매너

- **친근하고 따뜻한** 톤
- 예시 문구:
  - "환영합니다!"
  - "편하게 물어보세요"
  - "저장 완료!"
  - "멋진 안내서가 완성되었어요"

## 5. 기본 컴포넌트

### 5.1 버튼 (Button)

#### 스타일 변형

| 변형 | Primary | Secondary | Ghost |
|------|---------|-----------|-------|
| 기본 | 채움 (Primary) + 흰 텍스트 | 테두리 + Primary 텍스트 | Primary 텍스트만 |
| 호버 | 약간 어둡게 | 연한 배경 채움 | 밑줄 |
| 포커스 | 포커스 링 (Primary) | 포커스 링 | 포커스 링 |
| 비활성 | 50% 투명도 | 50% 투명도 | 50% 투명도 |
| 로딩 | 스피너 표시 | 스피너 표시 | 스피너 표시 |

#### 크기

| 크기 | 높이 | 패딩 | 폰트 | Tailwind |
|------|------|------|------|----------|
| Large | 48px | 24px | 16px | `h-12 px-6 text-base` |
| Medium | 40px | 16px | 14px | `h-10 px-4 text-sm` |
| Small | 32px | 12px | 12px | `h-8 px-3 text-xs` |

```tsx
// 버튼 컴포넌트 예시
<Button variant="primary" size="md">
  저장하기
</Button>
```

### 5.2 입력 필드 (Input)

| 상태 | 스타일 |
|------|--------|
| 기본 | 테두리 (`border`) + 흰 배경 |
| 포커스 | Primary 컬러 테두리 + 포커스 링 |
| 에러 | Error 컬러 테두리 + 에러 메시지 (아래) |
| 비활성 | 회색 배경, 입력 불가 |

```tsx
// 입력 필드 예시
<Input
  label="숙소명"
  placeholder="예: 제주 감성 펜션"
  error="숙소명을 입력해주세요"
/>
```

### 5.3 카드 (Card)

| 속성 | 값 | Tailwind |
|------|-----|----------|
| 배경 | Surface (`#FAF9F7`) | `bg-surface` |
| 테두리 | 1px (`#E8E4E0`) | `border border-border` |
| 모서리 | 12px | `rounded-xl` |
| 그림자 | 0 2px 8px rgba(0,0,0,0.05) | `shadow-sm` |
| 내부 여백 | 16px | `p-4` |

### 5.4 블록 (에디터용)

| 상태 | 스타일 |
|------|--------|
| 기본 | 흰 배경, 연한 테두리 |
| 호버 | 약간 밝은 배경, 테두리 강조 |
| 선택됨 | Primary 테두리, 연한 Primary 배경 |
| 드래그 중 | 그림자 증가, 약간 확대 |

### 5.5 Bottom Sheet (모바일)

| 속성 | 값 |
|------|-----|
| 배경 | 흰색 |
| 상단 핸들 | 40px 너비, 회색, 4px 높이 |
| 모서리 | 상단만 20px 라운드 |
| 최대 높이 | 화면의 90% |
| 오버레이 | rgba(0,0,0,0.3) |

## 6. 에디터 레이아웃

### 6.1 PC 레이아웃 (2-Column)

```
┌─────────────────────────────────────────────────────────┐
│ Header (로고, 저장, 발행)                                │
├────────────────────────┬────────────────────────────────┤
│                        │                                │
│   미리보기 패널         │   컨트롤 패널                   │
│   (스마트폰 목업)       │   (아코디언 메뉴)               │
│                        │                                │
│   ┌──────────────┐    │   ├─ 기본 정보                  │
│   │              │    │   ├─ 블록 관리                  │
│   │   안내서     │    │   ├─ 테마 설정                  │
│   │   미리보기   │    │   └─ AI 설정                   │
│   │              │    │                                │
│   └──────────────┘    │                                │
│                        │                                │
└────────────────────────┴────────────────────────────────┘
```

- 좌측: 40% (미리보기)
- 우측: 60% (컨트롤)
- 최소 너비: 1024px

### 6.2 모바일 레이아웃 (Full-Screen)

```
┌─────────────────────────┐
│ Header                  │
├─────────────────────────┤
│                         │
│   안내서 미리보기        │
│   (전체 화면)            │
│                         │
│   [블록 터치 → 편집]     │
│                         │
├─────────────────────────┤
│ [+ 블록] [테마] [미리보기]│ ← 하단 고정 버튼
└─────────────────────────┘

┌─────────────────────────┐
│ ▔▔▔                     │ ← Bottom Sheet
│ 블록 편집                │
│                         │
│ [저장] [취소]            │
└─────────────────────────┘
```

## 10. 애니메이션 & 트랜지션

### 10.1 기본 트랜지션

| 용도 | Duration | Easing | Tailwind |
|------|----------|--------|----------|
| 호버 | 150ms | ease-out | `transition-colors duration-150` |
| 모달/시트 | 200ms | ease-out | `transition-all duration-200` |
| 페이지 전환 | 300ms | ease-in-out | `transition-all duration-300` |

### 10.2 애니메이션 예시

```css
/* Bottom Sheet 올라오기 */
@keyframes slideUp {
  from { transform: translateY(100%); }
  to { transform: translateY(0); }
}

/* 블록 추가 */
@keyframes fadeIn {
  from { opacity: 0; transform: scale(0.95); }
  to { opacity: 1; transform: scale(1); }
}
```
