[사업 정의서] 스테이 가이드 (Stay Guide)1. 서비스 개요서비스명: 스테이 가이드 (Stay Guide)한 줄 정의: 앱 설치가 필요 없는, 숙박업소를 위한 AI 모바일 객실 안내서 (SaaS).핵심 컨셉: "모바일 청첩장의 호스피탈리티 버전" + "잠들지 않는 AI 컨시어지"플랫폼 형태: 모바일 웹 (Web-based), 반응형 웹사이트2. 타겟 고객 및 문제 정의 (Pain Point)A. 메인 타겟 (구매자)대상: 20~40대 감성 숙소, 에어비앤비 호스트, 펜션 사장님.문제점:반복되는 단순 문의(와이파이, 주차, 맛집 등)로 인한 피로감.종이 안내문은 훼손되기 쉽고, 정보 수정 시 재인쇄 비용 발생.노션(Notion)은 기능적이지만, 모바일에서 '감성'이 부족하고 가독성이 떨어짐.니즈: 내 숙소의 '감성'을 해치지 않으면서, 손님 문의를 자동으로 처리해 줄 도구가 필요함.B. 최종 사용자 (이용자)대상: 해당 숙소에 투숙하는 손님.문제점: 고작 1박을 위해 전용 앱을 설치하거나 회원가입 하는 것을 극도로 싫어함.니즈: 링크 하나로 모든 정보를 즉시 확인하고 싶음.3. 핵심 기능 (Key Features)A. 기본 기능: 모바일 가이드북 (The Guide)No-Install 접속: 별도 앱 설치 없이 URL 링크 하나로 접속.감성 UI 템플릿: 퍼블리셔의 감각을 살린 고퀄리티 디자인 테마 제공 (노션 대비 우위).실시간 정보 수정: 관리자 페이지에서 수정 시 즉시 반영 (종이 안내문의 단점 해결).필수 정보 블록화: 와이파이(복사 버튼), 주차장(지도 연동), 입퇴실 시간, 주변 맛집 등.B. 킬러 기능: AI 컨시어지 (The AI Chatbot)문맥 기반 답변: 사장님이 입력한 가이드북 데이터를 기반으로 AI가 답변.할루시네이션 방지: 입력되지 않은 정보에 대해서는 "사장님께 직접 문의해 주세요"라고 답변하도록 제어.24/7 응대: 심야 시간대 단순 문의(비번, 보일러 등) 자동 처리.퀵 버튼(Chips): 자주 묻는 질문(와이파이, 체크아웃 등)을 버튼 형태로 제공하여 타이핑 최소화.4. 서비스 흐름 (User Journey)[Step 1: 제작 및 구매 (사장님)]가입/제작: 웹사이트에서 기본 정보를 입력하고 미리보기(Watermark 포함) 생성.결제: 네이버 스마트 스토어에서 이용권 구매 (초기 PG 연동 부담 제거).인증: 구매 후 발송된 '시리얼 넘버'를 입력하여 워터마크 제거 및 정식 URL 생성.[Step 2: 전달 및 사용 (손님)]전달: 사장님이 예약 확정 문자/카톡에 stayguide.com/숙소명 링크 첨부.접속: 손님은 앱 설치 없이 링크 클릭.조회: 와이파이, 맛집 등 정보 확인.문의: 궁금한 점은 우측 하단 'AI에게 물어보기' 버튼 클릭 후 해결.5. 비즈니스 모델 (BM) 및 가격 정책A. 기본형 (Basic) - "종이 안내문 대체"가격: 연 3~5만 원 (또는 최초 1회 5만 원 + 연 유지비 1만 원)기능: 모바일 가이드북 생성, 무제한 수정, 기본 트래픽 제공.B. 고급형 (Pro) - "AI 비서 고용"가격: 월 구독형 (예: 월 9,900원 ~ 19,900원)기능: 기본형 기능 + AI 챗봇 기능 활성화.마진 구조: 저렴한 API(gpt-4o-mini 등) 사용하여 토큰 비용 최소화, 구독료로 수익 창출.6. 기술 스택 및 개발 전략 (Solo Dev)Front-end: React 또는 Next.js (초기 로딩 속도 최적화, SEO보다는 UX 중점).Back-end: Node.js (NestJS) 또는 Python (FastAPI). 가벼운 서버리스(Supabase 등) 권장.Database: PostgreSQL (Supabase 활용 추천 - RAG 구현을 위한 pgvector 지원).AI/LLM: OpenAI API (gpt-4o-mini 모델 사용 - 가성비 최우수).Hosting: Vercel (FE) + Supabase (BE/DB) 조합으로 초기 비용 $0 시작.7. 주요 리스크 및 대응 방안구분리스크대응 방안경쟁노션(Notion), 인스타 무료 사용압도적인 모바일 UI/UX 심미성 제공, AI 챗봇으로 기능적 우위 확보.기술AI의 거짓 답변 (환각)RAG(검색 증강 생성) 기술 적용 + 엄격한 시스템 프롬프트로 입력된 정보 외 답변 차단.비용API 토큰 비용 과다 발생질문 횟수 제한(Tier별 차등) 및 경량화 모델 사용. 답변 길이 제한.영업초기 고객 확보 어려움스마트 스토어 입점, 에어비앤비 호스트 커뮤니티 바이럴, 하단 'Made by StayGuide' 낙수 효과.
